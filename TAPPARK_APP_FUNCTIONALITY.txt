                    1. USER SCREENS & FUNCTIONALITY

1.1 SPLASH SCREEN (Splash1Screen)
   - Initial app loading screen
   - Displays while checking authentication status
   - Redirects to appropriate screen based on authentication state

1.2 GREETINGS SCREEN (GreetingsScreen)
   - Welcome screen for new users
   - Navigation to Login or Signup screens
   - First-time user introduction

1.3 LOGIN SCREEN (LoginScreen)
   - User authentication
   - Email and password input with validation
   - Password visibility toggle
   - Form validation (email format, required fields)
   - Error handling for invalid credentials
   - Network error handling
   - Automatic routing based on user type:
     * Attendant -> Attendant Dashboard
     * Subscriber -> HomeScreen (or AboutScreen if no vehicles)
     * Admin -> HomeScreen
   - Navigation to Signup screen
   - Back navigation to Greetings screen

1.4 SIGNUP SCREEN (SignupScreen)
   - New user registration
   - Fields: School ID, First Name, Last Name, Email, Password
   - Email format validation
   - Password strength validation (minimum 6 characters)
   - Form validation for all required fields
   - Account creation with backend API
   - Automatic login after successful registration
   - Navigation to AboutScreen for vehicle setup
   - Back navigation to Login screen

1.5 ABOUT SCREEN (AboutScreen)
   - App introduction and information
   - Description of TapPark system
   - "Add Vehicle" button (navigates to AddVehicleScreen)
   - "Skip for now" option (navigates to HomeScreen)
   - Shown to new users after registration

1.6 HOME SCREEN (HomeScreen)
   - Main dashboard for users
   - Profile picture display (with initials fallback)
   - Navigation drawer access
   
   REGISTERED VEHICLES SECTION:
   - Horizontal scrollable list of user's registered vehicles
   - Displays vehicle type, brand, plate number
   - Vehicle type icons (Car, Motorcycle, Bike/E-bike)
   - Empty state with "Add Vehicle" button if no vehicles
   - Tap vehicle card to book parking
   - Scroll indicators for multiple vehicles
   
   FREQUENTLY USED PARKING SPACE SECTION:
   - Horizontal scrollable list of frequently used parking spots
   - Shows location name, spot number, spot ID
   - Displays usage count
   - Shows current availability status
   - Time slot information
   - "BOOK" button for quick booking
   - Location logos (FPA, FU)
   - Empty state if no frequent spots
   - Scroll indicators for multiple spots
   
   SELECT PARKING AREA SECTION:
   - Horizontal scrollable parking area cards
   - FPA Underground Parking
   - FPA Round-About Parking
   - Main Campus Parking
   - Area logos and names
   
   BOOKING FLOW:
   - Select vehicle -> Select parking area -> Auto-assign spot -> Confirm booking
   - Vehicle type compatibility checking
   - Prevents multiple active bookings
   - Vehicle selection modal with compatibility filtering
   - Booking confirmation modal
   - Vehicle type mismatch error handling
   - Pre-booking status checks for occupied/reserved spots
   - User-friendly alerts for unavailable spots
   - Smooth animated navigation to ActiveParkingScreen after successful booking
   
   SEE ALL SPOTS MODAL:
   - Scrollable list of all parking spots in selected area
   - Color-coded spot cards by status:
     * Green: Available
     * Red: Occupied
     * Yellow/Orange: Reserved
     * Gray: Unknown
   - Displays spot number, type, and status
   - Area selection dropdown
   - Fetches all spots regardless of status
   
   PARKING LAYOUT MODAL:
   - Interactive SVG parking layout visualization
   - Tap parking area card to view layout
   - Color-coded spot overlays:
     * Green: Available
     * Red: Occupied
     * Yellow/Orange: Reserved (by other users)
     * Blue: User's booked spot
     * Gray: Unknown
   - Tap spot overlay to view spot details (status, type, section)
   - Responsive touchable overlays matching SVG spot positions
   - Zoom and scroll support
   - Spot status fetched in real-time

1.7 ACTIVE PARKING SCREEN (ActiveParkingScreen)
   - Three-tab interface: Parking Ticket, Parking Layout, Parking Time
   
   PARKING TICKET TAB:
   - QR Code display (for attendant scanning)
   - QR code contains: reservation ID, vehicle details, parking area, spot number
   - Parking details display:
     * Display Name
     * Parking Area
     * Date
     * Vehicle Details (Brand, Type)
     * Parking Slot (Number, Type)

   PARKING LAYOUT TAB:
   - Interactive SVG parking layout visualization
   - Pinch-to-zoom functionality
   - Pan and scroll support
   - Responsive touchable overlays on parking spots
   - Color-coded spot status indicators:
     * Green: Available
     * Red: Occupied
     * Yellow: Reserved (by other users)
     * Blue: User's booked spot
   - Spot details modal on tap
   - Refresh button to reload layout
   - Empty state if no layout available
   - Supports nested SVG groups (floors, sections)
   - Dynamic parsing for admin-added sections
   
   PARKING TIME TAB:
   - Circular timer display
   - Real-time elapsed time tracking
   - Progress ring animation
   - Time format: HH:MM:SS
   - Parking details card:
     * Vehicle information
     * Parking Area
     * Date
     * Plate Number
     * Parking Spot
     * Partial Amount
   
   REAL-TIME FEATURES:
   - Polling for session status updates (every 1 second)
   - Automatic timer start when attendant scans QR code
   - Timer syncs with backend start time
   - Session end detection
   - Parking end modal with:
     * Duration (HH:MM:SS format)
     * Hours deducted (HH.MM format, e.g., 1.30 for 1 hour 30 minutes)
     * Remaining balance (HH.MM format)
     * Penalty notice (if user exceeded balance)
     * Parking area and spot details

   - Penalty will be deducted from next subscription plan

     * START PARKING: Scan QR to start parking session
     * END PARKING: Scan QR to end parking session

2.3 ADMIN/ATTENDANT PARKING MANAGEMENT
   - Direct parking session management from dashboard
   - Available in slot details modal for admins/attendants
   - Features:
     * End Parking Session:
       - Available for occupied spots
       - Calculates duration and deducts hours (HH.MM format)
       - Applies penalty system if balance exceeded
       - Updates spot status to available
       - Records session completion
     * Cancel Booking:
       - Available for reserved spots
       - Cancels reservation without hour deduction
       - Updates spot status to available
       - Removes reservation record
     * Guest Booking:
       - Create bookings for temporary users (guests)
       - Guest user account automatically created
       - Vehicle registration for guest
       - Booking starts immediately (occupied status)
       - QR code generated for guest booking
       - Guest bookings tracked in system

6.4 TRANSACTION HISTORY
   - View all transactions
   - Transaction types:
     * Subscription purchases (hours added)
     * Parking sessions (hours deducted)
   - Transaction details:
     * Type
     * Amount (hours in HH.MM format)
     * Date and time
     * Status
     * Transaction ID
     * Additional details (hours used in HH.MM format)
   - Transaction filtering
   - Pagination support
   - Hours displayed in HH.MM format (e.g., 1.30 = 1 hour 30 minutes)

6.5 HOUR DEDUCTION
   - Automatic hour deduction on parking session end
   - Minute-precise deduction (HH.MM format, e.g., 1.30 for 1 hour 30 minutes)
   - Minimum 1 minute deduction for any parking session (even seconds)
   - Balance update after deduction
   - Transaction recording
   - Display format: Hours and minutes (e.g., 83.30 = 83 hours 30 minutes)
   
6.6 PENALTY SYSTEM
   - Detects when parking duration exceeds available subscription balance
   - Calculates penalty hours (excess duration beyond balance)
   - Records penalty in database for deduction from next subscription plan
   - Displays penalty notice in parking end modal (shows hours and minutes exceeded)
   - Applies regardless of remaining balance amount
   - Deducts only available hours from current subscription
   - Excess hours recorded as penalty for next subscription
   - Example: User has 1.00 hour, parks for 2.00 hours → 1.00 hour deducted, 1.00 hour penalty recorded

================================================================================
                        USER FLOWS
================================================================================

9.1 NEW USER FLOW
   1. Launch app -> Splash Screen
   2. Greetings Screen
   3. Signup Screen (register account)
   4. About Screen (app introduction)
   5. Add Vehicle Screen (register first vehicle)
   6. Home Screen (main interface)

9.2 EXISTING USER FLOW
   1. Launch app -> Splash Screen
   2. Login Screen (authenticate)
   3. Home Screen (or AboutScreen if no vehicles)

9.3 PARKING BOOKING FLOW
   1. Home Screen -> Select vehicle
   2. Select parking area
   3. Automatic spot assignment
   4. Booking confirmation
   5. Active Parking Screen
   6. Attendant scans QR code (starts session)
   7. Timer starts
   8. Attendant scans QR code again (ends session)
   9. Session end summary
   10. Return to Home Screen

9.4 ATTENDANT FLOW
   1. Login as Attendant
   2. Dashboard Screen (view parking slots)
   3. Options:
      a. QR Scanner Screen (scan user QR codes to start/end sessions)
      b. Direct Management (tap slot -> End Parking/Cancel Booking)
      c. Guest Booking (tap available slot -> Book for Guest)
   4. View activity history
   5. Manage settings

9.5 GUEST BOOKING FLOW (Admin/Attendant)
   1. Admin/Attendant taps available parking spot
   2. Guest booking modal opens
   3. Fill in guest information:
      * Guest Name (required)
      * Plate Number (required)
      * Vehicle Type (required)
      * Brand, Model, Color (optional)
   4. Submit booking
   5. System creates:
      * Temporary guest user account
      * Vehicle record for guest
      * Reservation (status: active, start_time: NOW())
   6. Spot status changes to occupied
   7. QR code generated for guest booking
   8. Booking confirmation displayed
