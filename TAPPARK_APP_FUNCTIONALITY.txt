================================================================================
                    1. USER SCREENS & FUNCTIONALITY
================================================================================

1.1 SPLASH SCREEN (Splash1Screen)
   - Initial app loading screen
   - Displays while checking authentication status
   - Redirects to appropriate screen based on authentication state

1.2 GREETINGS SCREEN (GreetingsScreen)
   - Welcome screen for new users
   - Navigation to Login or Signup screens
   - First-time user introduction

1.3 LOGIN SCREEN (LoginScreen)
   - User authentication
   - Email and password input with validation
   - Password visibility toggle
   - Form validation (email format, required fields)
   - Error handling for invalid credentials
   - Network error handling
   - Automatic routing based on user type:
     * Attendant -> Attendant Dashboard
     * Subscriber -> HomeScreen (or AboutScreen if no vehicles)
     * Admin -> HomeScreen
   - Navigation to Signup screen
   - Back navigation to Greetings screen

1.4 SIGNUP SCREEN (SignupScreen)
   - New user registration
   - Fields: School ID, First Name, Last Name, Email, Password
   - Email format validation
   - Password strength validation (minimum 6 characters)
   - Form validation for all required fields
   - Account creation with backend API
   - Automatic login after successful registration
   - Navigation to AboutScreen for vehicle setup
   - Back navigation to Login screen

1.5 ABOUT SCREEN (AboutScreen)
   - App introduction and information
   - Description of TapPark system
   - "Add Vehicle" button (navigates to AddVehicleScreen)
   - "Skip for now" option (navigates to HomeScreen)
   - Shown to new users after registration

1.6 HOME SCREEN (HomeScreen)
   - Main dashboard for users
   - Profile picture display (with initials fallback)
   - Navigation drawer access
   
   REGISTERED VEHICLES SECTION:
   - Horizontal scrollable list of user's registered vehicles
   - Displays vehicle type, brand, plate number
   - Vehicle type icons (Car, Motorcycle, Bike/E-bike)
   - Empty state with "Add Vehicle" button if no vehicles
   - Tap vehicle card to book parking
   - Scroll indicators for multiple vehicles
   
   FREQUENTLY USED PARKING SPACE SECTION:
   - Horizontal scrollable list of frequently used parking spots
   - Shows location name, spot number, spot ID
   - Displays usage count
   - Shows current availability status
   - Time slot information
   - "BOOK" button for quick booking
   - Location logos (FPA, FU)
   - Empty state if no frequent spots
   - Scroll indicators for multiple spots
   
   SELECT PARKING AREA SECTION:
   - Horizontal scrollable parking area cards
   - FPA Underground Parking
   - FPA Round-About Parking
   - Main Campus Parking
   - Area logos and names
   
   BOOKING FLOW:
   - Select vehicle -> Select parking area -> Auto-assign spot -> Confirm booking
   - Vehicle type compatibility checking
   - Prevents multiple active bookings
   - Vehicle selection modal with compatibility filtering
   - Booking confirmation modal
   - Vehicle type mismatch error handling
   - Navigation to ActiveParkingScreen after successful booking

1.7 ACTIVE PARKING SCREEN (ActiveParkingScreen)
   - Three-tab interface: Parking Ticket, Parking Layout, Parking Time
   
   PARKING TICKET TAB:
   - QR Code display (for attendant scanning)
   - QR code contains: reservation ID, vehicle details, parking area, spot number
   - Parking details display:
     * Display Name
     * Parking Area
     * Date
     * Vehicle Details (Brand, Type)
     * Parking Slot (Number, Type)
     * Plate Number
   - "Add to Favorites" button
   - Status indicator (Waiting for attendant / Active session)
   
   PARKING LAYOUT TAB:
   - Interactive SVG parking layout visualization
   - Pinch-to-zoom functionality
   - Pan and scroll support
   - Clickable parking spots
   - Highlights user's reserved spot
   - Spot details modal on tap
   - Refresh button to reload layout
   - Empty state if no layout available
   
   PARKING TIME TAB:
   - Circular timer display
   - Real-time elapsed time tracking
   - Progress ring animation
   - Time format: HH:MM:SS
   - Parking details card:
     * Vehicle information
     * Parking Area
     * Date
     * Plate Number
     * Parking Spot
     * Partial Amount
   
   REAL-TIME FEATURES:
   - Polling for session status updates (every 1 second)
   - Automatic timer start when attendant scans QR code
   - Timer syncs with backend start time
   - Session end detection
   - Parking end modal with:
     * Duration
     * Charge amount (hours deducted)
     * Remaining balance
     * Parking area and spot details
   - Automatic navigation to HomeScreen after session ends

1.8 PROFILE SCREEN (ProfileScreen)
   - User profile information display
   - Profile picture with upload/edit functionality
   - User name and email display
   - Menu options:
     * Change Password
     * Registered Vehicles
     * Terms & Conditions
     * Balance
     * FAQ
   - Profile picture options:
     * Take Photo (camera)
     * Choose from Library
     * Remove Picture
   - "How can we help you?" support button
   - Profile data loading from API
   - Profile picture caching and refresh

1.9 FAVORITES SCREEN (FavoritesScreen)
   - List of saved favorite parking spots
   - Displays for each favorite:
     * Parking area name
     * Spot number and type
     * Location address
     * Status (Available/Occupied)
   - "BOOK" button for quick booking
   - Remove from favorites (trash icon)
   - Empty state if no favorites
   - Vehicle selection modal for booking
   - Vehicle type compatibility checking
   - Navigation to ActiveParkingScreen after booking

1.10 HISTORY SCREEN (HistoryScreen)
   - Parking session history
   - Displays for each session:
     * Reservation ID
     * Location name
     * Scan timestamps (Start/End)
     * Duration
     * Hours deducted
     * Vehicle information (Type, Brand, Plate Number)
     * Spot number and type
     * Booking date
     * Status (Completed/Active/Reserved)
   - "Add to Favorites" button (heart icon)
   - "BOOK AGAIN" button for rebooking same spot
   - Tap card to view detailed reservation information
   - Reservation details modal with full information
   - Pagination support
   - Empty state if no history

1.11 BALANCE SCREEN (BalanceScreen)
   - User profile display
   - Debit card-style balance display
   - Student ID display
   - Total hours remaining display
   - "+ ADD HOURS" button (navigates to TopUpScreen)
   - Transaction history:
     * Transaction type (Subscription purchase / Parking session)
     * Amount (hours added or deducted)
     * Plan name (for subscriptions)
     * Date and time
     * Transaction icons
   - Tap transaction to view details
   - Transaction details modal with:
     * Transaction type
     * Plan name (if applicable)
     * Amount
     * Date
     * Status
     * Transaction ID
     * Additional details (location, spot, hours used, etc.)
   - Empty state if no transactions

1.12 TOP UP SCREEN (TopUpScreen)
   - User profile display
   - Subscription plans list
   - Plan details for each:
     * Plan name
     * Description
     * Cost (in pesos)
     * Number of hours
   - Plan selection
   - Confirmation modal:
     * Plan details
     * Total amount
     * Hours to be added
   - Purchase confirmation
   - Success message with:
     * Hours added
     * Total hours remaining
   - Navigation back to BalanceScreen after purchase

1.13 REGISTERED VEHICLES SCREEN (RegisteredVehiclesScreen)
   - User profile display
   - Vehicles grouped by type:
     * Registered Car
     * Registered Motorcycle
     * Registered Bike
   - For each vehicle displays:
     * Plate Number
     * Brand
     * Model
     * Color
     * Vehicle type icon
   - Delete vehicle button (trash icon)
   - Delete confirmation modal
   - Horizontal scrollable lists per vehicle type
   - Scroll indicators for multiple vehicles
   - "Add Vehicle" button
   - Empty state if no vehicles
   - Navigation to AddVehicleScreen

1.14 ADD VEHICLE SCREEN (AddVehicleScreen)
   - Vehicle registration form
   - Fields:
     * Vehicle Type (dropdown: Car, Motorcycle, Bike, E-bike)
     * Vehicle Color
     * Plate Number (required)
     * Vehicle Brand
     * Vehicle Model
   - Form validation
   - "Go Back" button
   - "+ Add" button
   - Success alert and navigation to HomeScreen
   - Animated car image with glow effects

1.15 CHANGE PASSWORD SCREEN (ChangePasswordScreen)
   - Password change form
   - Fields:
     * Current Password
     * New Password
     * Confirm Password
   - Password visibility toggles
   - Form validation
   - Password strength requirements
   - Success/error alerts
   - Navigation back to ProfileScreen

1.16 TERMS & CONDITIONS SCREEN (TermsAndConditionsScreen)
   - Terms of service display
   - Scrollable content
   - TapPark logo
   - Back navigation

1.17 FAQ SCREEN (FAQScreen)
   - Frequently Asked Questions
   - Expandable/collapsible FAQ items
   - Questions include:
     * What is TapPark?
     * How does TapPark Work?
     * What are the benefits?
     * Can I reserve parking in advance?
     * How does TapPark help with traffic?
     * What kind of technology does TapPark use?
     * How will I know if a parking spot is available?
     * And more...
   - Back navigation

                  2. ATTENDANT SCREENS & FUNCTIONALITY
================================================================================

2.1 DASHBOARD SCREEN (DashboardScreen)
   - Main attendant interface
   - Header with:
     * TapPark logo
     * Attendant name and profile picture
     * Activity history icon
     * Settings icon
   
   VEHICLE TYPES SECTION:
   - Cards for each vehicle type (Car, Motorcycle, Bike)
   - Displays for each type:
     * Vehicle type icon
     * Type name
     * Total Capacity
     * Occupied count
     * Available count
   - Real-time count updates
   
   PARKING SLOTS SECTION:
   - Grid or List view toggle
   - Vehicle type filter buttons (All, Car, Motorcycle, Bike)
   - Slots organized by Section
   - Each slot displays:
     * Slot ID
     * Vehicle Type
     * Status (Available/Occupied/Reserved)
     * Status color coding:
       - Green: Available
       - Red: Occupied
       - Yellow: Reserved
   - Tap slot to view details
   - Slot details modal with:
     * Slot ID
     * Vehicle Type
     * Status
     * Section
     * Parking Area
     * Address
   - Real-time polling (every 10 seconds) for status updates
   - Smooth animations for status changes
   - Status legend
   
   FLOATING ACTION BUTTON:
   - QR Scanner button (bottom right)
   - Navigates to QRScannerScreen
   
   ACTIVITY HISTORY MODAL:
   - Scan history table
   - Columns:
     * Scan Time
     * Action (Start/End)
     * Vehicle (Plate Number)
     * Parking Area
     * Spot
     * Status
   - Sortable columns
   - Search/filter functionality
   - Scrollable table
   
   SETTINGS MODAL:
   - Notification Preferences:
     * New Reservation Alerts toggle
   - Account Information:
     * Attendant ID
     * Attendant Name
     * Email
     * Assigned Area
   - Admin contact information
   - Change Password section (Admin only)
   - Log Out button

2.2 QR SCANNER SCREEN (QRScannerScreen)
   - Camera view with QR code scanning
   - Permission handling for camera access
   - Scanning overlay with frame
   - Animated scan line
   - Two scanning modes:
     * START PARKING: Scan QR to start parking session
     * END PARKING: Scan QR to end parking session
   - Status text display
   - Start Parking button (green)
   - End Parking button (red)
   - Timer display (when session active):
     * Circular progress timer
     * Elapsed time (HH:MM:SS)
     * Progress ring animation
   - Session info display:
     * Vehicle Plate
     * Spot Number
     * Reservation ID
   - QR code parsing and validation
   - API integration for:
     * Starting parking session
     * Ending parking session
   - Success/error alerts
   - Auto-stop scanning after 10 seconds
   - Close button to return to dashboard
